# Contributing to Sentry Span Optimizer

Thank you for your interest in contributing to Sentry Span Optimizer! This document provides guidelines and instructions for contributing.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/your-username/sample-rate-extension.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test your changes thoroughly
6. Commit your changes: `git commit -m "Add your meaningful commit message"`
7. Push to your fork: `git push origin feature/your-feature-name`
8. Open a Pull Request

## Development Setup

1. Install the extension in developer mode (see README.md for instructions)
2. Make changes to the source files
3. Reload the extension in your browser to test changes
4. Check the browser console for any errors

## Code Style

- Use consistent indentation (2 spaces)
- Add comments for complex logic
- Keep functions focused and small
- Use descriptive variable and function names

## Testing

Before submitting a PR, please:

1. Test the extension in Chrome
2. Test the extension in Firefox (if applicable)
3. Verify that all features work as expected
4. Check the browser console for errors
5. Test with different Sentry organizations/projects

## Pull Request Process

1. Update the README.md if needed
2. Update the version in manifest.json if making significant changes
3. Ensure your code follows the existing code style
4. Add comments for any complex logic
5. Test thoroughly before submitting

## Reporting Issues

When reporting bugs, please include:

- Browser and version
- Extension version
- Steps to reproduce
- Expected vs actual behavior
- Screenshots if applicable
- Console errors if any

## Feature Requests

Feature requests are welcome! Please open an issue describing:

- The feature you'd like to see
- Why it would be useful
- How it might work

Thank you for contributing!
